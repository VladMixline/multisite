# Настройка частых автоматических коммитов

## Описание

Система автоматических коммитов теперь поддерживает более частые проверки и коммиты изменений.

## Доступные скрипты

### 1. `auto-commit-frequent.ps1`
Проверяет изменения каждые 30 секунд и делает коммит, если прошло минимум 60 секунд с последнего коммита.

**Запуск:**
```powershell
.\scripts\auto-commit-frequent.ps1
```

### 2. `watch-and-commit.ps1`
Отслеживает изменения файлов в реальном времени с помощью FileSystemWatcher и делает коммит через 60 секунд после последнего изменения.

**Запуск:**
```powershell
.\scripts\watch-and-commit.ps1
```

### 3. `setup-frequent-commits.ps1`
Удобный скрипт для запуска системы частых коммитов в фоновом режиме.

**Запуск:**
```powershell
.\scripts\setup-frequent-commits.ps1
```

## Настройка

### Интервалы проверки

В файле `auto-commit-frequent.ps1` можно изменить:
- `$checkInterval` - интервал проверки изменений (по умолчанию 30 секунд)
- `$minTimeBetweenCommits` - минимальное время между коммитами (по умолчанию 60 секунд)

В файле `watch-and-commit.ps1` можно изменить:
- `$debounceTime` - время ожидания после последнего изменения перед коммитом (по умолчанию 60 секунд)

## Автозапуск

Для автоматического запуска при включении компьютера добавьте скрипт в планировщик заданий Windows или в автозагрузку.

## Остановка

Для остановки процесса частых коммитов:
1. Найдите процесс PowerShell в диспетчере задач
2. Или используйте: `Get-Process | Where-Object {$_.CommandLine -like "*auto-commit*"} | Stop-Process`
